Why rank-based tests work: deriving the Kruskal-Wallis test from scratch

statistics
hypothesis testing
A clinical trial comparing three pain management protocols motivates the Kruskal-Wallis test — a nonparametric alternative to ANOVA for skewed or ordinal data. This post derives the test statistic from first principles.
Author

Andreas Bogossian

Published

April 22, 2026

  • [INTRO] Set the scene: 45 post-surgical patients randomized to three pain management protocols — opioid-only, NSAID + nerve block, multimodal. 48 hours post-op, each rates pain on a 0–10 numeric rating scale (NRS). The clinical question: do the protocols produce different pain levels?
  • [INTRO] First instinct is ANOVA. But pain scores are bounded integers, distributions are right-skewed (most patients cluster at 2–4, a few report 8–10), and with 15 per group the CLT won’t save you. ANOVA’s p-value under these conditions is unreliable.
  • [INTRO] This is exactly the problem the Kruskal-Wallis test solves: replace raw scores with their ranks in the pooled dataset and test whether the ranks cluster by treatment. Under the null, every allocation of ranks to groups is equally likely — combinatorics, not distributional assumptions. This post derives the statistic from scratch.

When ANOVA breaks down

  • [TEXT] ANOVA compares between-group variance to within-group variance via the F-statistic, assuming residuals are approximately normal. With small samples and right-skewed data like these pain scores, that assumption fails and the nominal 5% significance level no longer corresponds to a 5% false-positive rate.
  • [TEXT] The simulation below makes this concrete. We generate data from three identical gamma distributions (null is true) and track how often ANOVA incorrectly rejects at α = 0.05, sweeping from heavy skew (similar to pain data) to near-normal. A calibrated test should sit at 5%. ANOVA drifts badly as skewness increases; KW stays controlled.

import numpy as np
from scipy import stats
from scipy.stats import f as f_dist

rng = np.random.default_rng(42)

N_PER_GROUP = 25
K = 3
N_SIM = 2000
ALPHA = 0.05
EFFECT = 0.6

shapes = [1.0, 3.0, 10.0, 50.0]
shape_labels = ["Very skewed (γ=1)", "Skewed (γ=3)", "Mildly skewed (γ=10)", "Near-normal (γ=50)"]


def make_groups(shape, rng_use, effect=EFFECT):
    # Scale so each group has unit variance regardless of shape
    scale = 1.0 / np.sqrt(shape)
    return [
        rng_use.gamma(shape, scale, N_PER_GROUP) + j * effect
        for j in range(K)
    ]


# Theoretical F(K-1, N-K) = F(2, 72) distribution for the null
df1 = K - 1
df2 = N_PER_GROUP * K - K
x_f = np.linspace(0, 10, 400)
f_theoretical = [{"x": float(x), "y": float(f_dist.pdf(x, df1, df2))} for x in x_f]
f_crit = float(f_dist.ppf(0.95, df1, df2))


scenarios = []

for shape, label in zip(shapes, shape_labels):
    # One representative dataset (with effect, for the interactive viz)
    groups_sim = make_groups(shape, rng)
    all_data = np.concatenate(groups_sim)
    ranks = stats.rankdata(all_data)
    group_ranks = [
        ranks[j * N_PER_GROUP : (j + 1) * N_PER_GROUP].tolist()
        for j in range(K)
    ]
    h_stat, p_kw = stats.kruskal(*groups_sim)
    f_stat, p_anova = stats.f_oneway(*groups_sim)

    # Null and power simulations
    reject_anova_null = 0
    reject_kw_null = 0
    reject_anova_effect = 0
    reject_kw_effect = 0
    f_stats_null = []

    for _ in range(N_SIM):
        sim_null = make_groups(shape, rng, effect=0.0)
        f_s, p_a = stats.f_oneway(*sim_null)
        _, p_k = stats.kruskal(*sim_null)
        f_stats_null.append(round(float(f_s), 4))
        if p_a < ALPHA:
            reject_anova_null += 1
        if p_k < ALPHA:
            reject_kw_null += 1

        sim_effect = make_groups(shape, rng, effect=EFFECT)
        _, p_a_eff = stats.f_oneway(*sim_effect)
        _, p_k_eff = stats.kruskal(*sim_effect)
        if p_a_eff < ALPHA:
            reject_anova_effect += 1
        if p_k_eff < ALPHA:
            reject_kw_effect += 1

    scenarios.append(
        {
            "shape": shape,
            "label": label,
            "groups": [g.tolist() for g in groups_sim],
            "ranks": group_ranks,
            "h_stat": round(float(h_stat), 3),
            "p_kw": round(float(p_kw), 4),
            "f_stat": round(float(f_stat), 3),
            "p_anova": round(float(p_anova), 4),
            "type1_anova": round(reject_anova_null / N_SIM, 3),
            "type1_kw": round(reject_kw_null / N_SIM, 3),
            "power_anova": round(reject_anova_effect / N_SIM, 3),
            "power_kw": round(reject_kw_effect / N_SIM, 3),
            "f_stats_null": f_stats_null,
        }
    )

The key insight: order is enough

  • [TEXT] To answer “does one protocol produce less pain than another?”, we don’t need to know how much less — only who ranks lower. The ordinal structure is sufficient. This is the motivation for throwing away raw scores and working with ranks instead.
  • [TEXT] Pool all 45 patient scores, sort them, assign rank 1 to the lowest, rank 45 to the highest. Now ask: do the low ranks cluster under one protocol, or are they randomly scattered? Under H_0 (all protocols equally effective), every allocation of the 45 ranks to three groups of 15 is equally likely — pure combinatorics. The original distribution’s shape is irrelevant.
  • [TEXT] There’s a broader pattern here. Spearman’s rank correlation does the same thing: replace two continuous variables with their ranks, then compute Pearson correlation. The rank transformation is a universal normalization trick — it strips away marginal distribution shape and leaves only ordinal structure.

Deriving the H statistic

Setting up the problem

We have K groups with n_1, \ldots, n_K observations and total N = \sum_j n_j. After pooling and ranking, let R_{ij} be the rank of the i-th observation in group j. We want to test whether the groups all share the same distribution.

Under H_0, the ranks 1, 2, \ldots, N are randomly distributed across the groups. The question is: what does a large deviation from this random distribution look like, and how do we measure it?

Expected rank mean under the null

Under H_0, every rank is equally likely to fall in any group. The expected value of a single rank drawn uniformly from \{1, \ldots, N\} is

\mathbb{E}[R] = \frac{1}{N} \sum_{r=1}^{N} r = \frac{N+1}{2}

So the expected mean rank in any group is \bar{R} = (N+1)/2, regardless of group size. If the null is true, all group mean ranks should hover around this value.

Building the test statistic

A natural measure of how far the group mean ranks deviate from the null expectation is a weighted sum of squared deviations, where each group contributes in proportion to its size:

T = \sum_{j=1}^{K} n_j \left(\bar{R}_j - \frac{N+1}{2}\right)^2

This has the same form as the between-group sum of squares in ANOVA, just applied to ranks.

The 12/N(N+1) scaling factor

To turn T into a statistic with a known null distribution, we need to normalize it by the variance of the ranks. For a discrete uniform distribution on \{1, \ldots, N\}, the variance is

\sigma_R^2 = \frac{N^2 - 1}{12} = \frac{(N-1)(N+1)}{12}

Dividing T by \sigma_R^2 / N (the approximate variance of a group mean rank for large equal groups) and simplifying gives the KW statistic

\boxed{H = \frac{12}{N(N+1)} \sum_{j=1}^{K} n_j \left(\bar{R}_j - \frac{N+1}{2}\right)^2}

  • [TEXT] The factor 12/[N(N+1)] scales H so that its expected value under H_0 equals K-1, which is exactly the degrees-of-freedom parameter of a chi-square distribution. For large N, H \xrightarrow{d} \chi^2(K-1) under H_0.
  • [TEXT] An equivalent and computationally cheaper formula rewrites the same quantity in terms of rank sums R_j = \sum_i R_{ij}:

H = \frac{12}{N(N+1)} \sum_{j=1}^{K} \frac{R_j^2}{n_j} - 3(N+1)

Both formulas give the same number; the second avoids computing each group’s mean rank explicitly.

Ties

  • [TEXT] When observations are tied, they receive the average of the ranks they would have occupied. For example, if two values share ranks 5 and 6, both get rank 5.5. Pain scores are integers from 0–10, so ties are common. With many ties a small correction is applied to H (Kruskal and Wallis 1952):

H_{\text{corrected}} = \frac{H}{1 - \dfrac{\sum_t (t^3 - t)}{N^3 - N}}

where the sum runs over all groups of tied observations and t is the size of each tied group. In practice this correction makes very little difference unless there is extreme tying.

NoteWhat KW doesn’t tell you

A significant KW result says something differs — not which pairs differ. For pairwise follow-up, use the Dunn test: pairwise rank-sum comparisons with a Bonferroni or Holm correction. In Python: scikit_posthocs.posthoc_dunn(groups, p_adjust='holm'). This is analogous to Tukey’s HSD after a significant ANOVA.

Kruskal-Wallis is ANOVA on ranks

There is a clean algebraic relationship between H and the F-statistic you would get from running ordinary one-way ANOVA directly on the ranks. If F_R denotes that F-statistic, then

F_R = \frac{H \, / \, (K-1)}{(N-1-H) \, / \, (N-K)}

  • [TEXT] This means KW and rank-ANOVA are monotonically related: they always agree on the ordering of datasets by evidence against the null. In large samples they give virtually identical p-values, since both converge to the chi-square distribution. They are not just analogous — they are algebraically the same procedure viewed through different lenses.
  • [TEXT] This is why KW is sometimes called “nonparametric ANOVA”: it takes the ANOVA logic — compare between-group variance to within-group variance — and applies it to a distribution that is known under the null (uniform ranks) instead of the assumed normal.

Why ranks normalize the distribution

  • [TEXT] The rank transformation replaces an unknown marginal distribution with the known uniform distribution. Whatever shape the original data has — exponential, log-normal, bounded integer like pain scores — after ranking, the pooled observations always fill positions 1 through N uniformly. The original scale and skewness are gone.
  • [TEXT] There is a formal result behind this. If X has any continuous distribution F, then F(X) \sim \text{Uniform}(0,1) — the probability integral transform. Ranks are an approximation to this: R_{(i)} / (N+1) \approx F(x_{(i)}). Rank-transforming is approximately applying the unknown CDF, which always produces uniform output.
  • [TEXT] The price is information loss. Ranks discard distances between values. For data that really is normal, this costs power: the asymptotic relative efficiency of KW vs ANOVA under normality is 3/\pi \approx 0.955. A 4.5% power penalty for insurance against non-normality is usually a good trade (Lehmann 2006). For pain scores — bounded, integer, skewed — there is no trade-off at all: ANOVA’s assumptions are violated and KW’s are not.

Code demo

Below we implement KW from scratch on the pain trial data to verify the formula, then compare it with the scipy.stats version.

Step-by-step implementation

import numpy as np
from scipy import stats


def kruskal_wallis(groups):
    N = sum(len(g) for g in groups)
    K = len(groups)

    all_data = np.concatenate(groups)
    ranks = stats.rankdata(all_data)

    idx = 0
    H = 0.0
    for g in groups:
        n_j = len(g)
        group_ranks = ranks[idx : idx + n_j]
        R_bar_j = group_ranks.mean()
        H += n_j * (R_bar_j - (N + 1) / 2) ** 2
        idx += n_j

    H *= 12 / (N * (N + 1))
    p_value = 1 - stats.chi2.cdf(H, df=K - 1)
    return H, p_value


H_manual, p_manual = kruskal_wallis(groups)
H_scipy, p_scipy = stats.kruskal(*groups)

print(f"Manual:  H = {H_manual:.4f},  p = {p_manual:.4f}")
print(f"SciPy:   H = {H_scipy:.4f},  p = {p_scipy:.4f}")
Manual:  H = 17.6147,  p = 0.0001
SciPy:   H = 18.2395,  p = 0.0001

Verifying the ANOVA-on-ranks equivalence

N = sum(len(g) for g in groups)
K = len(groups)

all_data = np.concatenate(groups)
ranks = stats.rankdata(all_data)

labels_arr = np.concatenate([
    np.full(len(g), j) for j, g in enumerate(groups)
])

F_R, p_F_R = stats.f_oneway(*[
    ranks[labels_arr == j] for j in range(K)
])

H_from_F = F_R * (K - 1) * (N - 1) / (F_R * (K - 1) + (N - K))
