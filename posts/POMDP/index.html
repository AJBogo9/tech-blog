The Core Problem

Imagine a robot navigating a corridor. It has a map but its GPS is broken — it only has a noisy distance sensor that tells it roughly how far the nearest wall is. The robot knows what actions it can take and what it might observe, but it never knows for certain where it is.

This is the essence of a Partially Observable Markov Decision Process (POMDP): the world has a true hidden state, but the agent only receives noisy, ambiguous observations. The agent must act despite this uncertainty.

If the robot could see its exact position, this would be a regular MDP — a much simpler problem. POMDPs are harder because the agent must reason about what it doesn’t know.

The POMDP Tuple

A POMDP is formally defined by the tuple (\mathcal{S}, \mathcal{A}, \mathcal{O}, T, \Omega, R, \gamma):

  • States \mathcal{S}: all possible configurations of the world
  • Actions \mathcal{A}: things the agent can do
  • Observations \mathcal{O}: what the agent can perceive (noisy, partial)
  • Transition model T(s' \mid s, a): how actions change the world
  • Observation model \Omega(o \mid s', a): how likely is observation o after arriving in state s' via action a
  • Reward R(s, a): scalar feedback signal
  • Discount \gamma \in [0,1): how much future rewards are down-weighted

The Belief State

Since the true state is hidden, the agent maintains a belief state b_t — a probability distribution over all possible states. b_t(s) is the agent’s current probability that the true state is s.

The belief state is everything the agent knows. Two agents with different histories but identical belief states should behave identically.

Belief Update

After taking action a and receiving observation o, the belief is updated via Bayes’ rule:

b_{t+1}(s') = \eta \cdot \Omega(o \mid s', a) \sum_{s \in \mathcal{S}} T(s' \mid s, a)\, b_t(s)

where \eta is a normalisation constant. In words:

  1. Predict: propagate the current belief through the transition model to get a prior over the next state
  2. Update: weight that prior by the likelihood of the observation — states that explain the observation well get higher probability
  3. Normalise: make it sum to 1

This is the same predict-update cycle as a Bayes filter or Kalman filter.

Interactive Demo: Robot in a 2D Grid

Below is a robot navigating a 6×6 grid with four symmetric obstacles. The robot has 4 binary sensors (N, S, E, W) that each report whether the adjacent cell is blocked — but each sensor flips independently with 10% probability. The blue heatmap shows the agent’s belief; the orange circle shows the robot’s true (hidden) position.

The grid is designed with deliberate 4-fold symmetry. The four central cells (2,2), (2,3), (3,2), (3,3) all have identical sensor signatures — a single reading can’t tell them apart. Navigate toward a corner or obstacle to break the symmetry and localise.

How to interact:

  • Press ↑ ↓ ← → arrow keys or click the buttons to move the robot
  • The robot moves with some noise (15% chance of slipping in place)
  • Watch the heatmap — it collapses when the robot reaches a uniquely identifiable position
stateLookup = new Map(free_cells.map(([r, c], i) => [r * COLS + c, i]))

function sampleObs(s_idx) {
  const [r, c] = free_cells[s_idx];
  const dirs = [[-1,0],[1,0],[0,1],[0,-1]]; // N, S, E, W
  let true_obs = 0;
  for (let d = 0; d < 4; d++) {
    const nr = r + dirs[d][0], nc = c + dirs[d][1];
    if (nr < 0 || nr >= ROWS || nc < 0 || nc >= COLS || grid[nr][nc] === 1)
      true_obs |= (8 >> d);
  }
  let obs = true_obs;
  for (let d = 0; d < 4; d++)
    if (Math.random() < 0.1) obs ^= (8 >> d);
  return obs;
}

function beliefUpdate(belief, T, obs) {
  const n = belief.length;
  const prior = Array(n).fill(0);
  for (let sp = 0; sp < n; sp++)
    for (let s = 0; s < n; s++)
      prior[sp] += T[sp][s] * belief[s];
  const posterior = prior.map((p, sp) => p * likelihood_table[sp][obs]);
  const Z = posterior.reduce((a, b) => a + b, 0);
  return Z === 0 ? belief : posterior.map(p => p / Z);
}

function doStep(state, action) {
  const Ts = { up: T_up, down: T_down, right: T_right, left: T_left };
  const deltas = { up: [-1,0], down: [1,0], right: [0,1], left: [0,-1] };
  const [dr, dc] = deltas[action];
  const [r, c] = free_cells[state.pos_idx];
  const nr = r + dr, nc = c + dc;
  const nextIdx = (nr >= 0 && nr < ROWS && nc >= 0 && nc < COLS)
    ? stateLookup.get(nr * COLS + nc)
    : undefined;
  let newPosIdx = state.pos_idx;
  if (nextIdx !== undefined && Math.random() < 0.85) newPosIdx = nextIdx;
  const obs = sampleObs(newPosIdx);
  const newBelief = beliefUpdate(state.belief, Ts[action], obs);
  const entropy = -newBelief.reduce((a, p) => a + (p > 1e-9 ? p * Math.log2(p) : 0), 0);
  return {
    pos_idx: newPosIdx,
    belief: newBelief,
    history: [...state.history, { step: state.step + 1, entropy }],
    last_obs: obs,
    last_action: action,
    step: state.step + 1,
  };
}

function makeInitialState() {
  const n = free_cells.length;
  const uniform = Array(n).fill(1 / n);
  const startIdx = stateLookup.get(2 * COLS + 2);
  const obs = sampleObs(startIdx);
  const posterior = uniform.map((p, s) => p * likelihood_table[s][obs]);
  const Z = posterior.reduce((a, b) => a + b, 0);
  const belief = posterior.map(p => p / Z);
  const entropy = -belief.reduce((a, p) => a + (p > 1e-9 ? p * Math.log2(p) : 0), 0);
  return { pos_idx: startIdx, belief, history: [{ step: 0, entropy }], last_obs: obs, last_action: null, step: 0 };
}

function beliefColor(p, maxP) {
  if (p === 0 || maxP === 0) return "#f1f5f9";
  const t = Math.pow(p / maxP, 0.5);
  return `rgb(${Math.round(219*(1-t)+30*t)},${Math.round(234*(1-t)+64*t)},${Math.round(254*(1-t)+175*t)})`;
}

Try navigating and notice:

  • Start: entropy drops from 5 bits (uniform over 32 cells) to ~2 bits — the robot is immediately narrowed to the 4 symmetric central cells
  • Central region: moving around the centre keeps ambiguity high; the 4 symmetric cells are still equally plausible
  • Near an obstacle: a new blocked sensor fires, ruling out most cells — belief sharpens
  • At a corner: entropy collapses to near 0 bits — corners have unique sensor signatures and fully localise the robot

Why Not Just Track the True State?

You might wonder: why not just have the robot track its exact position? In a real system you can’t — sensors are imperfect, motors slip, and the map may be wrong. The belief state is the correct answer to “what should the robot do given everything it has seen so far?”

Importantly, the belief state is sufficient: no matter how long the history is, all you need to plan optimally is b_t. This turns the POMDP into a (much larger) MDP over belief space, which is the foundation of POMDP solvers like PBVI and POMCP.

The Policy Problem

Given a belief state, what should the robot do? A policy \pi(b) maps belief states to actions. In a full POMDP solution, we compute the optimal policy by solving:

V^*(b) = \max_{a \in \mathcal{A}} \left[ \sum_{s} b(s)\, R(s,a) + \gamma \sum_{o \in \mathcal{O}} P(o \mid b, a)\, V^*(b') \right]

where b' is the belief after taking action a and receiving observation o. This is the Bellman equation over belief space.

Exact solutions are intractable for large state spaces (the belief space is continuous and infinite-dimensional), which is why approximate methods like Monte Carlo Tree Search in belief space (POMCP) are used in practice.

Summary

Concept What it means
Hidden state The world has a truth the agent can’t see directly
Observation Noisy, partial signal about the true state
Belief state Agent’s probability distribution over possible states
Belief update Bayes filter: predict via T, update via \Omega
Policy Maps belief → action; optimal policy solves Bellman over belief space

POMDPs are the principled framework for sequential decision-making under uncertainty. They appear in robotics (localization, planning), medical treatment (partially known patient state), and dialogue systems (unknown user intent). The belief update derived here is the core engine behind all of them.
